          success_count INTEGER DEFAULT 0,
          error_count INTEGER DEFAULT 0,
          total_requests INTEGER DEFAULT 0,
          last_used_at TEXT,
          last_error TEXT DEFAULT '',
          cooldown_until TEXT,
          tags TEXT DEFAULT '[]',
          remark TEXT DEFAULT '',
          source TEXT NOT NULL DEFAULT 'manual'

        CREATE INDEX IF NOT EXISTS idx_pool_status ON pool_accounts(status, deleted_at);
        CREATE INDEX IF NOT EXISTS idx_pool_provider ON pool_accounts(provider_type, deleted_at);

        CREATE TABLE IF NOT EXISTS pool_status_history (
          id INTEGER PRIMARY KEY AUTOINCREMENT,
          pool_account_id INTEGER NOT NULL,
          from_status TEXT NOT NULL,
          to_status TEXT NOT NULL,

          triggered_by TEXT NOT NULL DEFAULT 'manual',
          created_at TEXT DEFAULT (datetime('now')),
          FOREIGN KEY (pool_account_id) REFERENCES pool_accounts(id)

      `

    **Step 2: Create src/services/poolService.js**
    Class PoolService following GatewayDataService constructor(getDb) pattern:
    - Constructor: stores getDb function
    - _parseRow(row): parse credentials JSON, tags JSON, return new object (immutable)
    - getAll(filters = {}): SELECT with optional WHERE provider_type=? and/or status=?, WHERE deleted_at IS NULL, ORDER BY updated_at DESC. Map through _parseRow.
    - getById(id): SELECT single row by id WHERE deleted_at IS NULL, _parseRow or null
    - add(data): INSERT with provider_type, email, display_name, credentials (JSON.stringify), status='available', source='manual'. Return created account.
    - update(id, updates): Get existing, merge with spread (immutable), stringify credentials/tags if present, set updated_at. Return updated.
    - deleteAccount(id): UPDATE SET deleted_at = datetime('now'). Return true/false.
    - transitionStatus(accountId, newStatus, reason='', triggeredBy='manual'): Validate against VALID_TRANSITIONS[account.status]. Use db.transaction() for atomic UPDATE + INSERT into pool_status_history. Throw Error on invalid transition.
    - calculateHealthScore(account): successRate = total_requests > 0 ? (success_count / total_requests * 100) : 100. quotaScore = _getQuotaScore(account). health = round(0.7 * successRate + 0.3 * quotaScore). UPDATE health_score column. Return health.
    - _getQuotaScore(account): Check provider_quota_statuses table by looking up channel linked to this pool account. If no data, return 100. (Phase 2 default: always 100; Phase 5 will integrate with actual quota data.)
    - addApiKey(providerType, apiKey, baseUrl, displayName): Validate apiKey non-empty, no whitespace. Call add() with provider_type, credentials={api_key: apiKey, base_url: baseUrl || ''}, display_name. Return created.
    - enableAccount(id): Get account, verify status === 'disabled', call transitionStatus(id, 'available', 'Manual enable', 'manual').
    - disableAccount(id): Get account, call transitionStatus(id, 'disabled', 'Manual disable', 'manual').

    VALID_TRANSITIONS constant:
    ```javascript
    const VALID_TRANSITIONS = {

      cooldown:   ['available', 'error', 'disabled'],
      disabled:   ['available'],
    };
    ```
